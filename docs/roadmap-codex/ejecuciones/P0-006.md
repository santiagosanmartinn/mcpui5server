# Ejecucion P0-006 - validate_ui5_code v2

Fecha: 2026-03-10
Estado: completada

## Objetivo

Evolucionar `validate_ui5_code` a un motor v2 con reglas versionadas y separacion por categorias, manteniendo compatibilidad de salida previa.

## Cambios aplicados

1. Motor de validacion v2:
- Archivo: `src/utils/validator.js`
- Nuevo modelo:
  - `rulesVersion: "2.0.0"`
  - categorias: `structure`, `mvc`, `naming`, `performance`
  - soporte `sourceType`: `javascript`, `xml`, `auto`

2. Compatibilidad hacia atras:
- Se conserva `issues` con formato legacy:
  - `severity`
  - `code`
  - `message`
- Se agregan campos v2 sin romper consumo previo:
  - `issueDetails`
  - `issuesByCategory`
  - `rulesVersion`
  - `sourceType`

3. Tool actualizada:
- Archivo: `src/tools/ui5/validateUi5Code.js`
- Entrada extendida con `sourceType` opcional.
- Salida ampliada con metadata v2.
- `controllerMethods` y `missingLifecycleMethods` solo para `javascript`.

4. Integracion XML:
- Reutiliza `analyzeUi5Xml` para reglas de XML (parseo, handlers, senales de rendimiento).

## Pruebas agregadas/actualizadas

  - validacion de `rulesVersion`, categorias y flujo XML.
- `test/tools/validateUi5Code.tool.test.js`:
