# Ejecucion P0-004 - Escritura segura con preview + apply + rollback

Fecha: 2026-03-10
Estado: completada

## Objetivo

Incorporar escritura segura sobre archivos del workspace con tres capacidades:
- previsualizacion de cambios (`preview`),
- aplicacion controlada de patch (`apply`),
- restauracion por backup (`rollback`).

## Cambios aplicados

1. Nueva utilidad central:
- `src/utils/patchWriter.js`
- Funciones:
  - `previewFileWrite(...)`
  - `applyProjectPatch(...)`
  - `rollbackProjectPatch(...)`

2. Nuevas tools MCP en dominio `project`:
- `write_project_file_preview` (`src/tools/project/writePreview.js`)
- `apply_project_patch` (`src/tools/project/applyPatch.js`)
- `rollback_project_patch` (`src/tools/project/rollbackPatch.js`)

3. Registro de tools:
- `src/tools/index.js` actualizado.

4. Seguridad y robustez implementadas:
- Sandbox por `resolveWorkspacePath`.
- Limite de tamano por contenido (1,000,000 bytes por archivo).
- Validacion de hash base opcional (`expectedOldHash`) para evitar sobrescrituras inesperadas.
- Backups en `.mcp-backups/<patchId>.json`.
- Rollback idempotente por `patchId`.

5. Ajuste en busqueda de archivos:
- `src/utils/fileSystem.js` ignora `.mcp-backups` en `searchFiles`.

## Notas operativas

- `apply_project_patch` devuelve `patchId` solo cuando existen cambios reales.
- Si no hay cambios efectivos, devuelve `patchId: null` y lista de `skippedFiles`.
- `rollback_project_patch` sobre un patch ya revertido devuelve `alreadyRolledBack: true` sin volver a modificar archivos.
