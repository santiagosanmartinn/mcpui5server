# Ejecucion P0-003 - Parser XML UI5 para views/fragments

Fecha: 2026-03-10
Estado: completada

## Objetivo

Implementar una utilidad robusta para analizar XML UI5 (XMLView y Fragment), detectando namespaces, bindings y eventos, con errores tipificados.

## Cambios aplicados

1. Dependencia open source oficial agregada:
- `fast-xml-parser`

2. Nueva utilidad:
- Archivo: `src/utils/xmlParser.js`
- API principal:
  - `analyzeUi5Xml(code)`

3. Salida estructurada de la utilidad:
- `documentType` (`XMLView`, `Fragment`, `Unknown`)
- `rootTag`
- `namespaces`
- `controls` (tag, localName, namespacePrefix, path)
- `bindings` (attribute, expression, type, model, bindingPath, path)
- `events` (event, handler, tag, path)
- `models`

4. Manejo de errores tipificados:
- `INVALID_XML_INPUT` para entrada vacia/no valida.
- `UI5_XML_PARSE_ERROR` para XML invalido.
- `UI5_XML_ROOT_NOT_FOUND` cuando no hay raiz detectable.

## Pruebas agregadas

- Archivo: `test/utils/xmlParser.test.js`
- Casos cubiertos:
  1. XMLView con namespaces + bindings + eventos.
  2. FragmentDefinition con binding de lista y eventos.
  3. Binding complejo (`path: ...`) y expression binding (`{= ... }`).
  4. XML invalido con error `ToolError` y code estable.
