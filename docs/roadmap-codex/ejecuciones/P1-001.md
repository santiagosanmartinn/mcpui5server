# Ejecucion P1-001 - Sincronizador de manifest

Fecha: 2026-03-10
Estado: completada

## Objetivo

Agregar una tool para sincronizar `manifest.json` de forma idempotente en `models`, `routes` y `targets`, con validacion previa y posterior.

## Cambios aplicados

1. Utilidad dedicada:
- `src/utils/manifestSync.js`
- Funciones:
  - `validateManifestStructure(...)`
  - `synchronizeManifest(...)`

2. Tool MCP:
- `src/tools/project/syncManifest.js`
- Nombre: `sync_manifest_json`
- Capacidades:
  - `dryRun` (default true)
  - `apply` real (usa `apply_project_patch`)
  - preview hash/diff integrado
  - validacion pre y post parche

3. Registro de tool:
- `src/tools/index.js`

4. Pruebas:
- `test/utils/manifestSync.test.js`
- `test/tools/syncManifest.tool.test.js`

## Criterios de aceptacion P1-001

- Cambios idempotentes: OK
- Validacion previa y posterior: OK
- Integracion con patch seguro: OK

## Evidencia de validacion

- `npm run lint`: OK
- `npm run test:run`: OK (40 tests)
- `npm run check`: OK
- `npm run coverage`: OK

## Notas

- Si `dryRun: false` y no hay cambios efectivos, no se genera patch (`applyResult: null`).
- Si el manifest no pasa validacion estructural, la tool falla con error tipificado.
