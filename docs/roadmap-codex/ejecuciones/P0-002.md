# Ejecucion P0-002 - Parser AST para JS/TS

Fecha: 2026-03-10
Estado: completada

## Objetivo

Migrar el parser de analisis de archivos desde heuristicas regex a una base AST robusta para JavaScript y TypeScript, manteniendo compatibilidad de salida.

## Cambios aplicados

1. Dependencias oficiales open source agregadas:
- `@babel/parser`
- `@babel/traverse`

2. Refactor completo de parser:
- Archivo: `src/utils/parser.js`
- Implementacion AST para:
  - imports ESM
  - dependencias `sap.ui.define`
  - metodos de controller en `Controller.extend(...)`
  - metodos de clases `class ... extends Controller`
  - nombres de clase
  - deteccion de patron de controller

3. Robustez y compatibilidad:
- Parseo con `errorRecovery`.
- Fallback regex cuando AST no puede parsear codigo invalido.
- Contratos de salida preservados.

4. Adaptacion de consumidor principal:
- `src/tools/project/analyzeCurrentFile.js` actualizado para reflejar parser AST-backed.

## Pruebas agregadas/ajustadas

- `test/utils/parser.test.js`
  - soporte de TypeScript
  - metodos en clases ES6
  - casos invalidos con fallback estable
  - deteccion de patrones de controller
